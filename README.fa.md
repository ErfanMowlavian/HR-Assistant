# دستیار منابع انسانی — رتبه‌بندی رزومه به فارسی (MVP)

یک اپلیکیشن وب راست‌به‌چپ (RTL) فارسی که رزومه‌ها را بر اساس شرح شغل (Job Description) رتبه‌بندی می‌کند.
**هوش مصنوعی متن را می‌خواند** (فیلدهای ساختاریافته را استخراج می‌کند، هر مهارت را yes/partial/no قضاوت می‌کند)
و **ریاضیات قطعی رتبه‌بندی می‌کند** — یعنی قابل توضیح و شفاف است.

## موضوع پروژه

**سیستم رتبه‌بندی هوشمند رزومه به زبان فارسی (HR Assistant)** — یک اپلیکیشن وب راست‌به‌چپ که با استفاده از هوش مصنوعی، رزومه‌ها را بر اساس شرح شغل رتبه‌بندی می‌کند.

## سناریو

یک شرکت فناوری‌محور در حال استخدام، ده‌ها رزومه دریافت می‌کند. کارشناس منابع انسانی به جای خواندن دستی هر رزومه و حدس تطابق، از این سیستم استفاده می‌کند:
1. **ثبت شرح شغل** — کارشناس HR شرح شغل را وارد می‌کند
2. **استخراج خودکار** — هوش مصنوعی مهارت‌ها، سابقه کار، مدرک تحصیلی و سطح ارشدیت را استخراج می‌کند
3. **ارسال رزومه** — متقاضیان رزومه خود را ارسال می‌کنند (متن یا PDF)
4. **امتیازدهی هوشمند** — سیستم هر رزومه را در برابر شرح شغل امتیازدهی می‌کند
5. **رتبه‌بندی** — رزومه‌ها به ترتیب بهترین تطابق نمایش داده می‌شوند

## اهداف پروژه

| هدف | شرح |
|------|-------|
| **هدف کلی** | طراحی و پیاده‌سازی سامانه هوشمند رتبه‌بندی رزومه به زبان فارسی |
| **هدف اختصاصی ۱** | استخراج خودکار نیازمندی‌های شغلی از شرح شغل با استفاده از LLM |
| **هدف اختصاصی ۲** | دریافت رزومه متقاضیان (متن و PDF) و استخراج فیلدهای ساختاریافته |
| **هدف اختصاصی ۳** | امتیازدهی قطعی و شفاف به هر رزومه بر اساس تطابق با شرح شغل |
| **هدف اختصاصی ۴** | رتبه‌بندی رزومه‌ها با قابلیت توضیح‌دهی (Explainable Ranking) |
| **هدف اختصاصی ۵** | ارائه گزارش Gap به متقاضی |
| **هدف اختصاصی ۶** | ارزیابی کیفیت رتبه‌بندی با معیارهای Precision@3 و nDCG |

## جامعه هدف

| ذی‌نفع | نقش | نیاز اصلی |
|--------|------|----------|
| **کارشناس منابع انسانی (HR)** | کاربر اصلی سمت مدیریت | مشاهده رتبه‌بندی کاندیداها، بررسی جزئیات امتیاز، ویرایش نیازمندی‌ها |
| **متقاضی (Applicant)** | کاربر سمت متقاضی | ارسال رزومه، مشاهده گزارش Gap |
| **مدیر فناوری اطلاعات** | کارفرمای پروژه | استقرار سیستم، اطمینان از کیفیت |
| **توسعه‌دهندگان** | مجریان پروژه | نگهداری و توسعه سیستم |

## معماری

```
backend/   FastAPI + SQLite، لایه دروازه LLM، تست‌ها
frontend/  Next.js (App Router) + shadcn/ui، فارسی RTL، فونت وزیرمتن
```

## اجرا با Docker

```bash
docker compose up --build
```

- فرانت‌اند: http://localhost:3000
- بک‌اند + مستندات API: http://localhost:8000/docs

بارگذاری داده‌های دمو:
```bash
docker compose exec backend python -m app.seed
```
